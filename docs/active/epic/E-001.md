## Intent — the bigger-picture play

Stand up the thinnest end-to-end slice of the parlor: a pinned toolchain (flox + Vite + Svelte 5 + vitest + vite-plugin-singlefile) driven by justfile recipes (dev/test/check/build/deploy), a pure src/core/ seed (tile types, seeded RNG, 136-tile wall build) proven by the project's first property test, and a thin src/app/ Svelte 5 view that renders an empty table from a dealt wall — all compiling to one self-contained offline dist/index.html. No gameplay: the point is that the skeleton walks before any rules land, so P1's finishable hand, P3's single-file shippability, and P5's property-test rigor all have a live spine to grow along.

## Value to the design

Realizes the load-bearing architecture decisions as running code: the core/app split with zero DOM imports in core (the precondition for the action-log contract and all P5 property testing), seeded randomness (the precondition for deterministic simulation and attract mode), and the single-file offline compile target (P3's shippability made concrete on day one instead of retrofitted). The first passing property test (wall is exactly 136 tiles) establishes the testing idiom every later scoring/shanten property follows.

## Done looks like

From a fresh clone: `just test` passes with a vitest property test asserting the built wall is exactly 136 tiles; `just check` is clean; `just build` emits a single self-contained dist/index.html that, opened from the filesystem with no network, renders an empty table dealt from a seeded wall; `just dev` serves the same view locally. The repo has real src/core/ and src/app/ modules where today it has none.

## Context & constraints

Greenfield — the snapshot shows zero source modules, stories, or tickets, so this is the first epic on the board and everything else is downstream of it. Prerequisite (named as such, not hidden scope): the flox environment must be installable on the dev machine per CLAUDE.md's pinned-toolchain rule. In scope: toolchain pinning, justfile recipes including a deploy recipe, core seed (tiles/RNG/wall) with one property test, empty-table view, single-file build. Out of scope: all gameplay rules (deal-to-hand logic, draws, discards, calls, agari, shanten, yaku, scoring), AI bots, teaching UI, tile art beyond placeholder, service worker/manifest offline caching, and live CI/CD wiring to mahjong.b28.dev — the deploy recipe exists but standing up the Cloudflare pipeline is its own later play under P3. Respects all non-goals (no server, no accounts, no multiplayer). Right-size note (PE-7): this is at the upper edge of one ~2-hour block but is a single coherent play — thinning it further would leave a skeleton that doesn't walk end-to-end, which is the entire point of the signal.
