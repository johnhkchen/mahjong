## Intent — the bigger-picture play

Stand up the engine's spine in src/core/: partition the seeded 136-tile wall into live wall + 14-tile dead wall, flip the initial dora indicator with the indicator-to-dora mapping, and deal four 13-tile starting hands in E/S/W/N dealer order. Establish the hand-record type (seed + ordered action log) and the fold entrypoint (record in, table state out) as the engine's public contract. This is the moment the architecture's central claim — 'a hand is its record; table state is always derived by folding the pure engine' — becomes running, tested code rather than a document, and it is the substrate the turn loop, calls, replay, and review will all be written against.

## Value to the design

Advances P1 by producing the first legal slice of a finishable hand (a correct deal from a correct wall), and P5 by making the record/fold contract real and property-tested: deal conservation (4x13 hands + dead wall + live wall == 136, no duplicate tile ids), dora mapping over all 34 kinds including wraparound, same seed implies identical deal, and same record implies same folded state. Every subsequent epic gets cheaper and safer because it reads this contract instead of inventing state.

## Done looks like

From a seed, the engine deterministically produces a partitioned wall (live + 14-tile dead wall), a flipped dora indicator with correct indicator-to-dora mapping, and four 13-tile hands in E/S/W/N order; a hand record (seed + action log) folded through the public entrypoint yields that exact table state; the four named property tests pass under `just test`; and the app renders the dealt hand where the empty table used to be.

## Context & constraints

Grounded against a repo where src/** is empty: E-001 (done, archived) delivered the foundation — pinned toolchain, Vite + vite-plugin-singlefile single-file gate, vitest wiring, Cloudflare deploy — so this epic writes the first engine code on top of that scaffold; the E-001 toolchain is the prerequisite and it is already met. The record/fold contract must live in src/core/ with zero DOM/framework imports per the architectural invariants; the view change is limited to showing the dealt hand (Svelte 5 app shell may be minimal). Explicitly out of scope, deferred to later signals: draws, discards, the turn cycle, calls, agari, shanten, yaku, and scoring. No non-goal is touched — no server, no accounts, no multiplayer. The signal as pulled is right-sized to one epic (PE-7): wall partition + deal + record type is one coherent ~2-hour play, not several.
