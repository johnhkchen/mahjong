games are deterministic — submit errors with messages instead of word-of-mouth")._

## Intent — the bigger-picture play

Cash in the architecture's oldest promise. (1) Land the long-deferred **action-log text
notation** in core: a compact, versioned, human-readable serialization of a GameRecord
(header: version + game seed; one line per hand's action log) and its parser, with
round-trip property tests — parse(serialize(r)) folds deep-equal to r. This is the
public contract's text form, deferred since E-002; replay tooling and post-hand review
will read it later. (2) A "report bug" affordance in the app: a dialog with a freeform
message that auto-attaches the serialized record plus context (terminology, hand index,
action count), delivering via **copy-to-clipboard** and a **prefilled GitHub new-issue
URL** (johnhkchen/mahjong) — the no-server invariant holds because the reporter's own
browser does the delivery. (3) The owner's half: a paste-to-load entry that parses a
serialized record and folds it, reproducing the reported state exactly.

## Done looks like

Property tests round-trip arbitrary played GameRecords through the notation (serialize →
parse → fold deep-equal; malformed text fails loudly with line/position); the report
dialog copies a complete report (message + notation + context) and opens a prefilled
issue URL, falling back to clipboard-first instructions when the URL would exceed a safe
length (~6k chars); pasting a report's notation into the load entry reproduces the exact
table state; core stays DOM-free; single-file gate green; both terminologies covered.

## Context & constraints

The notation module is pure core (`src/core/`), zero DOM — it is the contract artifact,
the app feature is its first consumer. No server, no intake endpoint, no telemetry, no
accounts (invariants); GitHub-issue delivery is a plain URL the user opens. Versioned
header from day one — old reports must stay parseable. Out of scope: post-hand review
UI, replay stepping/scrubbing (future epics read this notation), automatic error
capture/global error handlers, screenshots.
