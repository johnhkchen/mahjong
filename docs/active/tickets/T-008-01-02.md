## Context

The han half: attach han values to the existing yaku-name catalog (open/closed variants
where they differ, e.g. sanshoku/ittsu/chanta drop one han open), and count dora + kan-dora
(all flipped indicators) into han. Dora never satisfies the one-yaku win gate by itself —
it prices a win, it cannot enable one.

_Advances: P5, P2_

## Acceptance Criteria

- [ ] Every name in the yaku catalog (incl. yakuman) has a han value with open/closed
  variants where standard rules differ, exported from core without changing yaku.ts's
  name-only API contract (a new scoring-side table maps names → han). Dora/kan-dora counts
  add han per copy held; a test proves a yakuless dora-laden hand still cannot win.
