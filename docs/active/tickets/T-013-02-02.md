## Context

The owner's half: a paste-to-load entry (inside the report dialog or a small dedicated
affordance — implementer's choice, documented) that parses a serialized report's
notation and replaces the app's GameRecord with it, folding to the exact reported
state — scores, hands, ponds, prompts, everything. Parse errors render the parser's
line/position message instead of loading. The existing `?seed=` boot pin stays; this is
its full-record big sibling.

_Advances: P5, P2_

## Acceptance Criteria

- [ ] An interaction test serializes a mid-game record, loads it through the paste
  entry, and asserts the folded table matches the original (deep-equal on the fold, and
  a DOM spot-check on hand + pond + scores); malformed paste shows the parse error and
  leaves the running game untouched; loading works with the game in any phase.
  `just test` green, single-file gate green.
